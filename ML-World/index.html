    Machine Learning Knowledge Graph

    1 minute read

    See more photos here

    Anat (my wife) told me that it would be nice to have a graphical representation of the different terms and domains in ML. I remember seeing static images of that type online, but I didn’t find a dynamic knowledge graph that gives a clearer picture for a selected term. I thought it would be interesting to implement one as a weekend project, and this post is the result of that attempt .

    Generally, the first step was to create (manually) this small file, based on Wikipedia. It consists of the main ML terms and their relations. I intend to extend it soon since the first version covers just the basics. In addition, I will happily expand/fix it based on any suggestion I’ll get in the comments .

    Then, I used javascript magic (mostly dark magic, here) to generate the graph, fuzzy search, and optional Wikipedia previews.

    The first impression you might get is -

    That’s because without selecting any term, you simply see the whole picture (no pun intended).

    Note that the javascript code doesn’t support touch screens at this point, so I would recommend viewing this graph on a device that has a touchpad/mouse.

    You have the following tools here that should help make the graph more usable:

    • Use the mouse-wheel to zoom in and out.
    • Click on any node to focus on the specific term - you’ll see its parents and direct children.
    • To move the whole graph - click on any location that is not a node and drag it.
    • Select the WikiPreview option to see a summary from Wikipedia when hovering above a node.
    • And last but not least, use the search box.
