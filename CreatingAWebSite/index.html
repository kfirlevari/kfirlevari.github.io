    In this post I’ll share how I built this website, i.e., one that is hosted in GitHub Pages, and powered by Jekyll & Minimal Mistakes.

    I think that it can be done by anyone that is enthusiastic about learning new technologies (actually, a keyboard, a screen, and a connection to the internet are also required ).

    I assume you have a basic familiarity with git. If you’re not sure how to work with git, you can start here.

    And most important, questions and comments are welcome

    Preparations

    Step 1. GitHub Pages

    Like I mentioned, the site is hosted in GitHub Pages. What does it mean? see here. A good starting point is to follow the instructions in that link, and to create your website repository (at <your github username>.github.io).

    Later you’ll place there the content of your website. For now, you should only be able to see there the “Hello World” page.

    Let’s also create a local clone of your website repository. E.g., to “/Repos/<your github username>.github.io” directory.

    Step 2. Preparations for Jekyll & Minimal Mistakes

    What’s Jekyll? a quote from here:

    “Jekyll is a static site generator. It takes text written in your favorite markup language and uses layouts to create a static website.”

    So basically, Jekyll can build a website from files that we write in a markup language. What markup language we’ll use? a simple one named Markdown.

    In order to run Jekyll, we need to prepare our environemnt. You should install the requirements listed here (i.e., Ruby, RubyGems, GCC, and Make). If you’re not sure how, then follow the installation guide that match your operating system here.

    What’s Minimal Mistakes? From here: “A flexible two-column Jekyll theme”. We’ll see later how to import it.

    Step 3. Source Directory Preparations

    Jekyll will transform a source directory into a website. In order to verify the changes that we do in the source directory while working on our website, and also to be resilient to failures (so that we won’t lose the source of our website if our computer crashes), we’ll use online version control.

    In this step, you should create a new repository for the source directory. For example - you can create another repository in github, it can be a private one this time (unlike <your github username>.github.io that needs to be public).

    After your created the source directory repository, clone it locally (e.g., to “/Repos/source” directory). We populate it in the next step.

    Step 4. Importing Minimal Mistakes

    Our goal in this step is to have these files and folders in your local source directory.

    One way to get these files is by cloning Minimal Mistakes repository locally (e.g., to “Repos/minimal-mistakes”), and then manually copying the required files to your “/Repos/source” directory. After this step, we won’t use “Repos/minimal-mistakes”, so you can delete it.

    Step 5. Populate the GemFile

    What is RubyGems (that we installed as part of Step 2)? From here: “RubyGems is a package manager for the Ruby programming language that provides a standard format for distributing Ruby programs and libraries (in a self-contained format called a "gem"), a tool designed to easily manage the installation of gems, and a server for distributing them.”

    We will populate the GemFile in “/Repos/source” with the packages that our website needs.

    Here is the GemFile I’m using:

    source "https://rubygems.org"

gem "github-pages", group: :jekyll_plugins
gem "jekyll-include-cache", group: :jekyll_plugins
gem 'jekyll-academicons-svg'
gem 'jemoji'
gem 'jekyll-scholar', group: :jekyll_plugins
    • github-pages, and jekyll-include-cache: required for us to be able to run our theme in GitHub Pages, as explained here.
    • jekyll-academicons-svg: in order to use academicons. Plugin repo here.
    • jemoji: because one can’t really express oneself online these days without emojis, or can we ? Plugin repo here.
    • jekyll-scholar: gives you citation super-powers, as explained here.

    Step 6. Install Jekyll and Bundler

    To install Jekyll and Bundler gems locally, run:

    gem install jekyll bundler

    We’ll use bundler to install gems and run Jekyll.

    Build Your Homepage

    Step 1. Set Repository Name

    The config file “_config.yml” contains many knobs, see here for the full list.

    You don’t have to update all of them right now, but you do need to set some of them -

    • Fill your website repository name, i.e, repository : “<your github username>.github.io”

    • Add “github: [metadata]” at the end of your config file (based on this).

    • Comment-out both “theme:” and “remote theme:” lines - we’re using a local copy of Minimal Mistakes, so we don’t need to import anything else.

    • Make sure that you have ‘jekyll-include-cache’ in the plugins array. Here is my plugins array (I think you’ll have the first 5):

    plugins:
  - jekyll-paginate
  - jekyll-sitemap
  - jekyll-gist
  - jekyll-feed
  - jekyll-include-cache
  - jemoji
  - jekyll-scholar

    Step 2. Sanity Check

    In your source directory, run:

    bundle

    and after all went well, run:

    bundle exec jekyll serve --incremental --verbose

    You should be able to see your new website at http://localhost:4000.

    Note that the optional –incremental flag enables you to test your website while changing it (but it’s experimental, see here), and the optional –verbose flag can help you debug issues you have in can Jekyll fails to build your site.

    Step 3. Design Your Website

    I’m no web designer, so I really don’t know what’s the right protocol here . But let me share a few things that I did and might help you, do them in any order you see fit:

    Config file

    Look closely at the configurations here, and modify your _config.yml accordingly.

    Navigation

    In _data/navigation.yml there’s the main array, based on it the links at the top are generated. E.g., my main array is:

    main:
  - title: "Blog"
    url: /blog/
  - title: "Projects"
    url: /projects/
  - title: "Publications"
    url: /publications/
  - title: "Past Teaching"
    url: /past-teaching/
  - title: "Personal"
    url: /personal/

    Pages, Posts, and Index

    • Create a new directory named “_pages” (see also here, and here). You should add an .md file in /_pages for every link that you added in the main array. E.g., I have /_pages/personal.md, that corresponds to the personal page.
    • If you want a blog, create a new directory named “_posts” (see also here, and here). Follow the name format when creating new posts. Note that only posts in the past are visible by default in the blog.
    • Your front page is generated based on index.html file.

    Front Matter and Layouts

    At the top of every page there’s a YAML front matter block. Read about it here. For example, the front matter of this page is:

    ---
layout: single
title:  "DIY - Homepage with Jekyll & Minimal Mistakes in GitHub."
toc: true
toc_label: "Steps"
toc_icon: "shoe-prints"
toc_sticky: true
---

    (“toc” stands for Table Of Content )

    You set the layout of the page by changing these values (e.g., the layout of this page is “single”). For all Minimal Mistakes layouts, see here.

    Note that we copied Minimal Mistakes source code to our project folder (our source directory), this means that you can modify every template that you want, just be sure that you know what you’re doing

    • E.g., the “single” layout file is found at _layouts/single.html.

    Publications

    I followed the instructions in jekyll-scholar and pascalpoizat.github.io.

    Generally, I did the following:

    • Created /assets/bibliography directory
    • Placed there a bib style file (*.csl). Chose your favorite, or modify one, there are too many out there. I used this style.
    • Also placed there references.bib file with all the bibtex citations.
    • In the config file, created this array:
    scholar:
  source: /assets/bibliography
  style: /assets/bibliography/mystyle.csl
    • Created the following publications.md file in /_pages/ :
    ---
permalink: /publications/
title: "Publications"
layout: single
classes: wide
header:
  overlay_image: /assets/images/Publications.jpg
  overlay_filter: 0.2
  caption: "[See more photos here](/personal/#photography)"
---

{ % bibliography % }

---

    Note that the header part relates to the header image, and the classes is for setting a wide screen. Also you shouldn’t have a space between { and %, or % and }.

    Version Control

    Just a reminder - if you followed my advice and created your source directory as a git repository, don’t forget to use git! Do small deltas of commit + push (e.g., once you’re done editing something at some page, push it to the remote source repository).

    You don’t want to work on something, delete it by mistake, and then find out that there’s no way to resotre it .

    Publish Your Homepage

    Once you’re satisfied with your website, i.e., you added content and even checked all the links and such, you probably want to publish it, right? .

    • Note that you can view it as if you’re using another device type. See instructions for Chrome here, for Safari here

    You should build your site with

    bundle exec jekyll build

    as instructed here. Note that if you’re running “bundle exec jekyll serve”, the site that is created isn’t the same one as the one created with build.

    Then, copy the generated site to your local /Repos/< your github username >.github.io directory, and push it to github. Good luck .
